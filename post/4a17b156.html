UnityUI的使用

Created2023-02-27|Updated2023-03-14
|Post View:

Unity UI

1、初步认识

Canvas组件

Canvas-Overlay模式

Pixel Perpect:高像素清晰度渲染。

Sort Order：多Canvas时，用来控制哪个画布在前面显示。

Canvas-Camera模式

Plane Distance：用来调节画布与摄像机贴合距离

Order in Layer:Sorting Layer（排序层）和Order in Layer相同时，距离相近越近，越靠前。

Sorting Layers如下：

双相机叠加

  1. 建立Camera
  2. 将UI相机拖给Canvas
  3. 将UI的相机设置为使用深度值填充，UI相机的深度要高于主相机
  4. 将主相机不渲染UI元素，将UI相机只渲染UI元素（Culling Mask为UI模式）

Canvas-World模式

Canvas以3D物体的形式显示在场景中。

Canvas缩放

​

​ 该组件会拉伸所有Canvas下的子元素

Constant Pixel Size

默认模式，默认静态拉伸。

Sacle With Screen Size（常用）

根据UI设计师给的图片进行适配：

根据宽度或者高度进行自动拉伸，自动计算。

一定要恢复原始大小 Set Native Size！！！

计算方法：18:9下：缺失两边，（设计分辨率高度 / 9）* 18

​ 16:9下：美术一般都会按照16:9出图

​ 4:3下：缺失上下，（设计分辨率宽度 / 4）* 3

Constant Physical Size

物理模式，根据英寸进行计算，不常用。

RectTransform组件

​ 出现位置：出现在所有Canvas子元素中，所有UI元素都应该有RectTransform组件

Pos（XYZ）：位置（X，Y，Z），单位（像素），位置是由轴点到锚点的距离

Width，Height：宽和高，单位（像素）

Anchors：锚点（四叶草），以父物体参考

这里X和Y的值为物体锚点（四叶草）到轴点（中心点）的距离

改成Pivot模式，可以改动轴点。

UGUI先进方式：双锚点

shift 快捷调节锚点，alt快捷移动

单锚点：距离计算是锚点到轴点的XYZ偏移

​ 双锚点：元素边界到锚线的距离，轴点到锚线的距离

​ 四锚点：元素的四个边界到锚框的距离

​ 锚点快速选择工具（十字符号）

​ 默认：移动锚点

​ Alt：移动锚点+快速停靠位置

​ Shift：移动锚点+移动轴点+快速停靠位置

元素渲染顺序

​ 以Hierarchy参考

下方物体在上方物体前显示

子物体在父物体前显示

下方物体永远在前显示，无论上方的层次结构

2、UGUI的基础组件

Panel组件

一个完整页面一个Panel

Panel和Canvas一样大，Canvas和屏幕一样大。（四锚点）

Text组件

文本内容

​ 字库选择

​ 字体样式（标准，粗体，斜体，粗斜体）

​ 字号

​ 行间距

​ 是否开启富文本

​ 加粗：文字

斜体：文字

大小：<size=字号>文字

修改颜色：<color=颜色名>文字

​ <color=#颜色数（十六进制）>文字

对齐方式（上下对齐，左右对齐）

几何对齐（参考字占用的几何空间）

水平溢出（折行，溢出[出右边框]）

垂直溢出（截断不显示，溢出[出下边框]）

​ 字号自适应 best fit

​ 关闭：使用原始字号

​ 开启：字号会在最小值和最大值之间自动调整

​ 字体颜色

阴影

外发光

Image组件

Color：用于混色，美术有时提供白图和色号，程序员对图处理，染成想要的颜色，可以减少包体大小。

四种类型图片

（1）普通模式

Preserve Aspect：保持图片的宽高比，高度或宽度自适应

Set Native Size：可以快速恢复美术提供的图片原始像素尺

（2）裁剪模式

Split Editor：进行拉伸

1,3,7,9区域不拉伸

2,8横向拉伸

4,6纵向拉伸

（3）瓦片模式（无缝贴图）

Tiled模式

（4）填充模式

Fill Method：填充方式（360度，水平和垂直）

​ Fill Origin：填充起始点

​ Fill Amount：填充百分比

​ Clockwise：顺时针或逆时针

​ Preserve Aspect：保持图片的宽高比，高度或宽度自适应

​ Set Native Size：可以快速恢复美术提供的图片原始像素尺寸

clockwise：控制顺逆时针

Raw Image组件

​ 原始图片组件：可以显示精灵或纹理

​ 功能相对于Image少，所以性能更好

​ 可以控制UV的偏移，来显示精灵的一部分

3、UI的事件交互

事件的三个要素：

是否忽略Canvas反向的事件操作

遮挡事件的物体（2D物体[精灵片]，3D物体）

遮挡事件的物体所在的渲染层

Raycast Target：是否接收Canvas的射线

1、button组件

按钮是否可交互

​ 按钮状切换效果

​ 动画效果影响的UI元素（Image和Text派生自Graphic）

​ 效果使用的图片

​ 普通状态下的颜色

​ 高亮状态下的颜色（鼠标移入）

​ 按下状态下的颜色（按钮按下）

​ 失活状态下的颜色（不可交互）

​ 颜色倍数（RGBA分量乘以倍数）

​ 切换动画持续时间

​ 是否开启键盘导航（Visualize开启后，能够看到导航线）

​ Navigation可以选择

​ 横向移动

​ 纵向移动

​ 自动匹配移动

​ 指定上下左右移动

​ 手机游戏可以关闭导航
